Wait — `NotificationHub` is a `record`, so it's immutable. I need to replace it in the list. But records use value equality so `Remove` might not find the right one. Let me adjust.

Actually, since `NotificationHub` is a `sealed record` (primary constructor), it has value equality. If I have `new NotificationHub(...)` it's the same as any other with the same field values. But we need reference identity. Let me add an `Id` field.

Actually, for owned entity collections with shadow keys, EF Core tracks them by reference. The `Remove` method on `List<T>` uses the default equality comparer. For records, that's value equality. Let me use `RemoveAll` with a predicate instead.

Better approach: keep `List<NotificationHub>` and use `_notificationHubs.RemoveAll(h => h.HubType == hubType && h.Identifier == identifier)` or use an index-based approach.
